# Project Charter

## Problem Statement

Setting up spec-driven development with Claude Code requires creating many files manually: slash commands, CLAUDE.md rules, agent rules, hook scripts, directory structure. This is tedious, error-prone, and means teams skip it or do it inconsistently.

## Goals

1. One command installs the full momentum toolkit into any project
2. Any developer or agent can understand the workflow by reading a single README
3. The system works for any tech stack, any language, any project size
4. The momentum repo itself demonstrates the workflow it provides

## Non-Goals

- Not a project generator (that's `/start-project`)
- Not a CI/CD tool
- Not tied to any specific framework or language
- Not a replacement for superpowers (complementary, not competing)

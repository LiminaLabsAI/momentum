# VAL-002 — Antigravity live evidence (Phase 18 G4)

Captured: 2026-06-15
Antigravity desktop app: `/Applications/Antigravity IDE.app` present
Antigravity CLI (`agy`): **does not exist as a standalone binary**.

## Scope

VAL-002 was opened post-v0.19.0 (Phase 16 Adapter Rework) and reopened
through Phase 18 G4 as the live-dogfood gate for the Antigravity swarm
parity. This evidence file documents what could be verified at the
desktop-app level and the structural reason VAL-002 cannot be fully
closed via CLI today.

## Key finding — no standalone CLI binary

The Antigravity desktop app ships as an IDE-only product. Comprehensive
search of the installed `/Applications/Antigravity IDE.app/` bundle
(and the related `/Applications/Antigravity.app/`) finds NO `agy` or
`antigravity-cli` executable:

```
$ find "/Applications/Antigravity IDE.app" -maxdepth 7 -type f \
       \( -name "agy" -o -name "antigravity-cli" \)
(no matches)

$ find "/Applications/Antigravity IDE.app/Contents/Resources/app/extensions/antigravity/bin/" \
       -maxdepth 2 -type f
language_server_macos_arm
sandbox-wrapper.LICENSE
sandbox-wrapper.sh

$ npm view agy
agy@0.0.0 | ISC | deps: none | versions: 1
.unpackedSize: 217 B               <-- placeholder package, not the real CLI

$ npm view @google/antigravity
404 — not found
```

The shipped binaries inside the IDE bundle are `language_server` and
`sandbox-wrapper` — the IDE's internal runtime, not a user-facing CLI.

Consequence: the momentum CLI's `adapters/antigravity/adapter.js::spawn()`
shells `agy --cwd <repoPath> --skill swarm-supervisor`, but `agy` is not
on PATH because the binary does not exist as a standalone form. This is
a structural limit of the Antigravity product surface, not a momentum
bug — momentum's spawn is correct for any future Antigravity CLI form.

## Check 3 — history reminder fires ✅ PASS (after live-payload fix)

FIRST ATTEMPT FAILED and exposed a real bug the synthetic tests missed:
`tool.execute.after` does NOT carry tool args in the live runtime (only the
before hook sees them), so the reminder never resolved the edited path.
Fixed with callID correlation (args captured at before-time, consumed at
after-time; bounded map; output.args fallback retained). After the fix, a
real write-tool call creating src/reminder-check.js produced
`.momentum/history-reminder-stamp` on disk. Unit tests updated to the
live-confirmed payload shape.

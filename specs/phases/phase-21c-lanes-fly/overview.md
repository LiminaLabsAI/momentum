# Phase 21c — Lanes Fly (One Recursive Wave Planner)

> **Target release:** v0.25.0
> **Arc:** final sub-phase of the Parallel Lanes family (21a Walk ✓ → 21b
> Run ✓ → **21c Fly**) — `specs/planning/platform-parallel-lanes.md`.
> **Stacked lane:** branch `phase-21c-lanes-fly` on `phase-21b-lanes-run`
> (releases land in order once the operator approves).
> **Planned:** 2026-07-03 under the operator's standing family directive;
> scope operator-committed 2026-07-02 (FEAT-028 + roadmap row).

## Goal

ONE wave engine at every scale of the plan graph
(ecosystem ⊃ repos ⊃ phases ⊃ tasks): dependency annotations on task
groups and phases, waves computed per level, swarm rewired as the
engine's top-scale consumer with byte-stable behavior — enabling the
target UX: N ideas → N planned lanes → governed parallel execution →
sequenced landings.

## Scope

### In (FEAT-028)

- **`core/waves/lib/waves.js`** — the generic engine: Kahn layering over
  `{from depends-on to}` edges, lexicographic stability inside a wave,
  cycle/self-dependency errors with clear participant lists. Extracted
  from (and error-string-compatible with) swarm's `wave-ordering.js`.
- **Swarm rewire** — `core/swarm/lib/wave-ordering.js` becomes a thin
  adapter over the core engine. Byte-stable: identical waves, identical
  error messages; the full swarm suite + e2e scenarios stay green
  unchanged.
- **Dependency annotations**:
  - task groups: `## Group N — title (deps: G0, G1)` heading suffix in
    `tasks.md`;
  - phases: optional `"deps": ["phase-…"]` array on
    `specs/phases/index.json` entries (deps on `complete` phases count
    as satisfied).
- **`momentum waves` CLI** — task scale (`--tasks`, default = the phase
  bound to the current branch per Rule 15; done groups drop out) and
  phase scale (default; non-complete phases), rendering waves plus
  suggested `momentum lanes open` commands for wave 1; `--json` unstable.
- **3-ideas e2e demo** — three dependency-annotated phases → computed
  waves → lanes opened per wave → sequenced landings through the 21b
  queue; automated test + captured evidence (evidence generated by a
  one-shot script, NOT rewritten by `npm test` — the TD-006 lesson).
- **Lane-state contract decision** reviewed at close (the arc's one-way
  door).

### Out (non-goals)

- Auto-spawning agent sessions per lane (execution engine stays parked
  with its written triggers).
- Cross-repo waves beyond what swarm already does (the rewire changes
  the engine, not swarm's surface).
- Publishing the lane-state format (see decision at close).

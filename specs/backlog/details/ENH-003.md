# ENH-003: Full Project Scaffold in `momentum init`

## Problem

`momentum init` currently installs only the tooling layer:
- `.claude/commands/` — 8 slash commands
- `.claude/settings.json` — hook config
- `.agent/rules/project.md` — agent rules
- `scripts/check-history-reminder.sh` — hook script

After install, a user cannot run `/brainstorm-phase` or `/start-phase` because both read `specs/status.md` as their first step — which doesn't exist. The project is not usable until Claude Code runs `/brainstorm-project` to scaffold the rest.

This creates a bad first-run experience: the success message suggests commands that immediately fail.

## Proposed Solution

Add spec template files to `core/specs-templates/` and have `momentum init` copy them into the target project, producing a fully usable project structure in one command.

### Files to add to `core/specs-templates/`

```
core/
  specs-templates/
    CLAUDE.md                          ← generic rules template (project name = placeholder)
    README.md                          ← generic README template
    specs/
      README.md
      status.md
      backlog/
        backlog.md
        details/.gitkeep
      changelog/
        .gitkeep                       ← empty; first entry created by /start-phase
      decisions/
        README.md
        0000-template.md
        impact-map.json                ← empty topics map
      phases/
        README.md
        index.json                     ← empty phases map
      planning/
        roadmap.md
      vision/
        project-charter.md
        principles.md
        success-criteria.md
```

### What gets copied to the target project

```
<target>/
  CLAUDE.md                  ← from core/specs-templates/CLAUDE.md
  README.md                  ← from core/specs-templates/README.md (if not exists)
  specs/                     ← full tree from core/specs-templates/specs/
  .claude/commands/          ← existing (from core/commands/)
  .claude/settings.json      ← existing (from adapters/claude-code/)
  .agent/rules/project.md    ← existing (from core/agent-rules/)
  scripts/                   ← existing (from core/scripts/)
```

### Skip-if-exists behaviour

Files that are project-specific and should not be overwritten:
- `CLAUDE.md` — warn if exists, skip
- `specs/status.md` — warn if exists, skip
- `specs/backlog/backlog.md` — warn if exists, skip

Files that are safe to overwrite (tooling, not data):
- All files in `.claude/commands/` — always overwrite (template updates)
- `specs/decisions/0000-template.md` — always overwrite
- `specs/decisions/impact-map.json` — skip if exists (user may have customised)

### Updated success message

```
✓ momentum installed successfully.

Next steps:

  New project:
    Open Claude Code and run: /brainstorm-project

  Existing project — plan your next phase:
    Open Claude Code and run: /brainstorm-phase
```

## Template Content Notes

- `CLAUDE.md` template: use `<Project Name>` as placeholder; user fills in after install
- `specs/status.md` template: `Current Phase: Phase 0 — Bootstrap (not started)`
- `specs/phases/index.json`: empty `phases: {}` object
- `specs/decisions/impact-map.json`: empty `topics: {}` object
- All other templates: copy verbatim from bootstrap plan v5.0

# ENH-008: `--coding-agent` Flag Naming

## Problem

The flag is currently designed as:

```bash
momentum init --coding-agent claude-code
momentum init --coding-agent cursor
```

The term "coding-agent" is a mouthful and may not be the right word. It's also ambiguous — are we talking about the AI model, the editor, the tool, or something else?

## Alternatives Considered

| Option | Example | Notes |
|--------|---------|-------|
| `--coding-agent` (current) | `--coding-agent cursor` | Accurate but verbose |
| `--agent` | `--agent cursor` | Shorter; still clear in context |
| `--tool` | `--tool cursor` | Generic; could mean many things |
| `--for` | `--for cursor` | Reads naturally ("install for cursor") |
| `--target` | `--target cursor` | Unambiguous but technical |
| `--adapter` | `--adapter cursor` | Exposes internal implementation detail |

## Decision Criteria

- Should be short and intuitive for a first-time user
- Should not leak implementation concepts (avoid `--adapter`)
- Should read naturally in a `--help` output
- Must be consistent with how we describe the concept in docs and README

## Recommendation

Decide before implementing TD-002 in Phase 3 so the flag lands with the right name from day one. Renaming after publish is a breaking change.

Leading candidates: `--agent` or `--for`.

## Impact

- `bin/momentum.js` — flag parsing
- `--help` output
- README / docs
- Any adapter.js discovery logic that reads the flag value

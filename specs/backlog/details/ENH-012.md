# ENH-012: Enhanced Rule 8 — Detailed History Logging

## Context

cerebrio-sapience's Rule 8 includes improvements over momentum's template that help agents log history more consistently:

### 1. "What counts as meaningful" list

Momentum's Rule 8 says "append meaningful changes" but doesn't define meaningful. cerebrio-sapience lists explicit triggers:
- An ADR was created or its status/decision changed → `[DECISION]`
- Phase scope was added to or reduced → `[SCOPE_CHANGE]`
- A bug, tech debt, or enhancement was added to backlog → `[DISCOVERY]`
- A new feature was added to the phase plan → `[FEATURE]`
- An architectural pattern or integration approach changed → `[ARCH_CHANGE]`
- A locked evaluator was defined or its evaluation set changed → `[EVALUATOR]`

### 2. `[EVALUATOR]` entry type

Momentum has `[NOTE]` (good catch, added in v0.5.0) but not `[EVALUATOR]`. Both should exist.

### 3. Impact-map.json reminder

After writing a history entry, the agent should check `specs/decisions/impact-map.json` and add any new topics. This ensures `/sync-docs` can find all affected files.

### 4. Hook script reminder reference

Reference to `scripts/check-history-reminder.sh` as a safety net.

## Implementation

- Expand Rule 8 in CLAUDE.md template with the "what counts as meaningful" list
- Add `[EVALUATOR]` to entry types (alongside existing `[NOTE]`)
- Add impact-map.json check step
- Add hook script reference

## Acceptance Criteria

- [ ] Rule 8 lists explicit triggers for each entry type
- [ ] `[EVALUATOR]` type documented
- [ ] Impact-map.json reminder included

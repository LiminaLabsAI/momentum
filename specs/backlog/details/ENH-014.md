# ENH-014: Multi-Repo Support in Rule 9

## Context

Projects with multiple repos (monorepo + libraries) need safeguards in the doc sync protocol to prevent accidentally modifying docs owned by other repos.

cerebrio-sapience developed these safeguards for Rule 9:

```markdown
- **Cross-repo docs**: NEVER update docs in another repo during sync.
  If a change affects a cross-repo doc, log it in history with
  `Affects-specs: ../repo/path/to/file.md` and flag it to the user.
```

And in `sync-docs.md`:
```
- NEVER update docs in another repo — only sync docs THIS repo owns
- If a history entry has `Affects-specs: ../other-repo/...`, flag it to the user
```

## Implementation

- Add cross-repo safeguard to Rule 9 in CLAUDE.md template (conditional: "if project has cross-repo dependencies")
- Add cross-repo safeguard to `sync-docs.md` command
- Consider adding a `specs/dependencies.md` template for multi-repo projects
- Consider a `--multi-repo` flag or auto-detection (presence of `specs/dependencies.md`)

## Acceptance Criteria

- [ ] Rule 9 includes cross-repo doc sync safeguard
- [ ] `sync-docs.md` includes cross-repo flag-to-user step
- [ ] Multi-repo projects can declare dependencies

# TD-002: Add `--coding-agent` Flag to `bin/momentum.js`

## Problem

`install.sh` supports `--coding-agent <name>` (default: `claude-code`). The Node.js CLI (`bin/momentum.js`) hardcodes `claude-code` — there is no flag to select a different adapter. 

When Phase 3 adds Cursor, Gemini CLI, or other adapters, users of the npx CLI will have no way to select them. The install.sh path would still work, but the primary install path (npx) would be stuck on Claude Code only.

## Proposed API

```bash
momentum init [target-dir] [--coding-agent <name>]
```

Examples:
```bash
npx @avinash-singh-io/momentum init
npx @avinash-singh-io/momentum init ./my-project --coding-agent cursor
npx @avinash-singh-io/momentum init --coding-agent gemini
```

Default: `--coding-agent claude-code` (no breaking change)

## Implementation

### Arg parsing (in `bin/momentum.js`)

```js
// parse --coding-agent <name> from process.argv
let codingAgent = 'claude-code';
const agentFlagIdx = args.indexOf('--coding-agent');
if (agentFlagIdx !== -1) {
  codingAgent = args[agentFlagIdx + 1];
  args.splice(agentFlagIdx, 2);
}
```

### Adapter resolution

```js
function init(targetDir, codingAgent) {
  const adapterDir = path.join(__dirname, '..', 'adapters', codingAgent);
  if (!fs.existsSync(adapterDir)) {
    console.error(`Error: Unknown coding agent '${codingAgent}'.`);
    console.error(`No adapter found at: ${adapterDir}`);
    process.exit(1);
  }
  // ... rest of install using adapterDir
}
```

### Adapter-specific install logic

Currently, `bin/momentum.js` hardcodes the Claude Code install steps. With the flag, the adapter should define its own install behaviour — mirroring how `adapter.sh` works for bash.

**Options:**
- **Option A**: Each adapter ships an `adapter.js` (Node.js equivalent of `adapter.sh`) with a `runInstall(targetDir, srcDir)` function — sourced by the CLI
- **Option B**: The CLI has an if/else per adapter (simpler, but doesn't scale)
- **Recommended**: Option A — consistent with the Phase 1 adapter.sh pattern; clean DIP boundary

### Update `--help` output

```
Options:
  --coding-agent <name>    Coding agent to install for (default: claude-code)
                           Available: claude-code
```

## Dependencies

- Requires Phase 3 to ship at least one additional adapter before the flag is useful
- Option A requires each adapter to add `adapter.js` alongside the existing `adapter.sh`
- Bump version to v0.4.0 on release (alongside ENH-003)

# ENH-009: Distribution Strategy

## Problem

Momentum currently ships via:
1. `install.sh` — bash-based, clone + run
2. `npx @avinash-singh-io/momentum init` — Node.js CLI, zero install

Both work, but neither is the "native" install path for any specific coding agent. Each agent ecosystem has its own plugin/extension model:

| Agent | Native Extension Model |
|-------|----------------------|
| Claude Code | MCP servers, or Claude Code extensions (marketplace TBD) |
| Cursor | Cursor Rules (`.cursor/rules/`), potentially Cursor Extensions |
| VS Code Copilot | `.github/copilot-instructions.md`, VS Code extensions |
| Gemini CLI | `GEMINI.md` convention, no formal plugin system yet |

## Options

### Option A: npx only (current)
- One command, zero install, works today
- Not discoverable via agent marketplaces
- Requires Node.js

### Option B: Native plugin per agent
- Claude Code: publish as a Claude Code extension (if/when marketplace opens)
- Cursor: distribute as a Cursor Rule Pack
- Each agent: follow their native install convention
- High discoverability, but high maintenance (N distributions per release)

### Option C: npx + native plugins (hybrid)
- Keep npx as the universal fallback
- Add native plugin distributions as agents' ecosystems mature
- Recommended long-term approach

## Decision Criteria

Before deciding, answer:
1. Does Claude Code have an official extension/plugin marketplace?
2. What is the Cursor Rule Pack distribution model?
3. Is there a meaningful discoverability advantage to native plugins vs npx?
4. Can we automate multi-target publishing (one release → npx + all agent plugins)?

## Recommendation

Defer until Phase 4+ when at least one additional adapter (Cursor or Gemini) is shipped. At that point, evaluate native distribution for that adapter's ecosystem as a proof of concept. If viable, generalize the pattern.

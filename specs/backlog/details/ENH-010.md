# ENH-010: CLAUDE.md Upgrade-Safe Architecture

## Problem

`momentum init` and future `momentum upgrade` replace CLAUDE.md entirely, but projects add project-specific content to CLAUDE.md (navigation tables, extra rules, cross-repo references). Upgrades clobber these customizations.

## Proposed Solution

Split CLAUDE.md into two zones using a marker:

```markdown
# Project Rules: <Project Name>

<!-- BEGIN MOMENTUM MANAGED -->
## Navigation (generic)
## Autonomous Behaviors (Rules 1-N)
## Naming Conventions
## Constraints
<!-- END MOMENTUM MANAGED -->

## Project Extensions
### Extended Navigation
### Additional Rules
### Cross-Repo Configuration
```

`momentum init` writes the full template on first install.
`momentum upgrade` replaces only the content between markers, preserving everything below `## Project Extensions`.

## Implementation Notes

- `bin/momentum.js` needs an `upgrade` subcommand (or enhance `init` with `--upgrade` flag)
- Marker-based replacement in `adapters/claude-code/adapter.js`
- If no markers found (pre-ENH-010 project), treat entire file as project-specific — back up and write fresh with markers, append old content under `## Project Extensions`
- Same pattern for `.agent/rules/project.md`

## Acceptance Criteria

- [ ] `momentum init` writes CLAUDE.md with markers on fresh install
- [ ] `momentum upgrade` replaces managed section, preserves extensions
- [ ] Pre-marker projects get safe migration (backup + append)
- [ ] `momentum upgrade` on a project with no local extensions works cleanly

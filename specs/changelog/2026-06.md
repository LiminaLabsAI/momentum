- **Phase 17 — Swarm: Single-Session Multi-Project Feature Delivery** brainstormed
  on branch `phase-17-swarm`. Target v0.20.0; Claude Code only (Codex + Antigravity
  parity deferred to Phase 18). Replaces queued Reach (Cursor + Gemini → shifts
  to Phase 19). User-articulated gap: today's ecosystem + orchestration
  (Phase 9–15) supports cross-repo awareness but not SUSTAINED PARALLEL work
  across multiple repos in one session. Scout fetches snapshots; dispatch
  fans out one prompt and synthesizes; handoff transfers control. None of
  them sustain a multi-hour multi-repo feature delivery.

  Capability shape: **swarm**. The user's session is the **conductor**.
  Per-repo **supervisor** subagents spawn pinned to each impacted repo's cwd,
  each running momentum's normal `/start-phase` → implement → `/sync-docs`
  → `/complete-phase` loop INSIDE its repo. Conductor coordinates waves
  (computed from `ecosystem.json` dependency edges), surfaces inbox
  decisions, broadcasts cross-cutting concerns, synthesizes at fan-in.

  Architecture: file-based coordination, no daemon, no server. State lives
  at `<eco>/swarms/<id>/` (manifest.json + board.json materialized cache +
  contracts + inbox + signals + tokens + details + changes). Agents
  stateless; swarm survives session boundaries via disk reconstitution.

  Indexing strategy: 4 layered (board cache + git SHA invalidation +
  incremental session-log reader + supervisor context isolation). Token-cost
  analysis projects 95% saving vs. naive file-reading (conductor turn ~3KB
  vs ~290KB; full swarm ~3M tokens / ~$2-5 vs ~60M / ~$50-100).

  Modes: autopilot / checkpoint (default) / interactive. v0.20.0 ships
  autopilot + checkpoint; interactive deferred.

  Intervention surface: 8 patterns identified, 5 ship in v0.20.0 (plan
  approval, wave checkpoint, mid-flight question via inbox, context push
  via `/swarm tell`, broadcast via `/swarm broadcast`). Defer to v0.20.x:
  discuss thread, manual takeover (pause/resume), rewind.

  Portability (split a swarm into a focused side-session; join independent
  sessions into one swarm; converge multiple sessions to one) schema hooks
  baked into v0.20.0 manifest; commands ship in Phase 17.5 v0.20.1 — no
  schema migration needed.

  Per-repo spec discipline preserved: phase brief gains optional
  `swarm:` + `wave:` + `initiative:` frontmatter; solo and swarm-member
  phases are byte-shape-compatible. Existing `/start-phase`, `/complete-phase`,
  `/validate` unchanged in behavior; extended to handle the new fields.

  End-to-end validation: 3 synthetic ecosystem fixtures (3-repo linear,
  4-repo branched, 5-repo wide) at `tests/fixtures/swarm-ecosystems/`.
  Reproducible. Cerebrio bootstrap deliberately deferred to post-release
  user activity (real projects stay untouched during phase work).

  Research: 7-agent workflow on 2026-06-12 surveyed industry patterns
  (Anthropic multi-agent research system, AWS Labs CAO, Devin 2, Cursor 3,
  Claude Code worktrees, CrewAI hierarchical, OpenAI Swarm, LangGraph
  supervisor, Pact, Saga) and converged on conductor + supervisor +
  files-as-channels.

  Roadmap renumber: Phase 17 = Swarm (Claude Code, v0.20.0); Phase 17.5 =
  Swarm Portability (v0.20.1); Phase 18 = Swarm Parity Codex + Antigravity
  (v0.20.2); Phase 19 = Reach (v0.21.0); Phase 20 = Intelligence (v0.22.0);
  Phase 21 = Platform (v1.0).

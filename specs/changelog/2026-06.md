- **Phase 11 brainstormed — Dynamic Orchestration & Context Handover.**
  Three primitives (`scout`, `dispatch`, `handoff`) ship equally
  weighted in v0.14.0 with three invocation doors (slash command + NL
  inference + CLI) over one shared `core/orchestration/` library.
  Visibility = live `▸` narration + persistent log via single event
  stream. Handoff pickup = SessionStart auto-greet with `[y/skip]`
  confirm PLUS explicit `/continue` + `momentum continue`, layered.
  Dispatch returns synthesis + collapsible per-repo + run artifact,
  synchronous. Tracking contract: cheap layer (session log + run
  artifact + handoff inbox) auto every time; curated layer
  (history.md / backlog.md) auto only when meaningful by Rule 3
  thresholds; no new history entry types. Adapter fallback = graceful
  degradation with CLI as universal floor, every degraded mode
  labeled. ENH-023 + ENH-024 (capability flag type unification) ride
  this phase in Group 0. Aggressive non-goals (no streaming dispatch,
  no new entry types, no ADR auto-creation, no multi-machine handoff)
  hold scope for a single-phase release. Scaffold at
  `specs/phases/phase-11-orchestration-handover/`. Commit `014f1a6`
  on branch `phase-11-orchestration-handover`.

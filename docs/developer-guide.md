# Developer Guide

## Contributing to momentum

This repo uses its own toolkit. Everything you need is already installed.

## Setup

```bash
git clone https://github.com/avinash-singh-io/momentum
cd momentum
# momentum's own commands are in .claude/commands/
```

### Working on a non-POSIX filesystem (external drive, exFAT, NTFS)

If you cloned this repo onto a filesystem that doesn't preserve POSIX permissions (e.g. an external USB-C drive formatted exFAT/NTFS, or some network mounts), git may report every file as modified with `100644 → 100755` mode flips. To suppress this, set `core.fileMode` locally for this clone:

```bash
git config core.fileMode false
```

This is a **per-clone** setting (`.git/config`) — it is not committed and will not affect other contributors.

macOS may also create AppleDouble `._*` sidecar files for every tracked file. These are filtered by the repo's root `.gitignore`. If you ever see them committed (`git status` shows `._foo.md` as untracked or staged), the gitignore is misconfigured — file an issue.

## Development Workflow

1. Check current state: read `specs/status.md`
2. Plan a phase: `/brainstorm-phase`
3. Start: `/start-phase`
4. Track discoveries: `/track` or `/log`
5. Complete: `/sync-docs` then `/complete-phase`

### Brainstorm Gate (since v0.8.0)

All three brainstorm-style commands (`/brainstorm-phase`, `/brainstorm-idea`, `/start-project`) honor the **Brainstorm Gate Contract**: they draft in conversation only and never write to `specs/` until you explicitly approve. The gate is enforced two ways:

- **Discipline**: each command's markdown declares the contract with a red-flag table.
- **Hook**: `scripts/brainstorm-gate.sh` is registered as a Claude Code `PreToolUse` hook. While `.momentum/brainstorm-active` exists, the hook blocks any `Write`/`Edit`/`MultiEdit` whose target lies under `specs/`.

If you ever see "Brainstorm gate active" in stderr unexpectedly, check whether a previous session left a stale sentinel:

```bash
ls -la .momentum/brainstorm-active && rm .momentum/brainstorm-active
```

### Autonomous Execution Contract (since v0.8.0, engine v0.9.0)

`/start-phase` declares the Autonomous Execution Contract: once a plan is approved, the engine runs all groups end-to-end with one hard stop — at the merge + release gate after the final group's verification passes. Phase 7a ships the contract (spec only); Phase 7b ships the engine that obeys it. Until 7b lands, the contract is the model's operating instructions.

## Repo Layout

- `core/` — tool-agnostic logic (commands, instructions, scripts, specs-templates)
- `adapters/<agent>/` — per-agent overlays (e.g. `adapters/claude-code/commands/review-code.md`, `adapters/claude-code/scripts/brainstorm-gate.sh`)
- `bin/momentum.js` — zero-dependency Node.js CLI
- `tests/` — Node's `node:test` runner; serial via `--test-concurrency=1`

When editing a command in `.claude/commands/` for local testing, mirror the same change in `core/commands/` (or `adapters/<agent>/commands/` if it's agent-specific). The `.claude/commands/` content is generated by `momentum init` / `momentum upgrade` — it is not the source of truth.

## Testing

```bash
npm test
```

Runs the full `tests/*.test.js` suite (currently 58 tests via Node `node:test`). The glob `tests/*.test.js` (not `tests/`) avoids loading macOS AppleDouble `._*.test.js` files.

For a manual smoke test of a fresh install:

```bash
rm -rf /tmp/momentum-smoke
node bin/momentum.js init /tmp/momentum-smoke --agent claude-code
ls /tmp/momentum-smoke/.claude/commands/
ls /tmp/momentum-smoke/scripts/
```

You should see all slash commands and `brainstorm-gate.sh` + `check-history-reminder.sh`.
